사용자 인증 처리 서버

요구사항
1. 클라이언트로 부터 이메일과 패스워드를 받는다.
2. 이메일과 패스워드를 데이터베이스에 저장된 데이터와 비교한다.
3. 비교한 값이 동일하면 인증 토큰을 생성하고, 요청 결과로 클라이언트에 돌려준다.
4. 인증결과를 캐시서버에 저장한다. 포함되는 정보는 발급시간, 만료시간, 사용자 정보다.
5. 캐시서버에 저장된 인증 정보는 3시간 동안 유효하며 3시간이 지난 뒤 자동으로 폐기된다.
5-1. 클라이언트가 폐기를 요청하면 만료시간을 넘기지 못하더라도 인증 토큰을 폐기 가능해야한다.
5-1. 이전에 발급받은 인증 토큰이 아직 사용 가능한 상태인지 확인할 수 있어야한다.


서비스 정의
1. 사용자 인증 토큰 발급 서비스 
2. 사용자 인증 토큰 만료 서비스
3. 사용자 인증 토큰 검증 서비스

RESTful URL 정의

/users                  GET     사용자 정보조회
/tokens/<사용자번호>       POST    사용자 인증 토큰 발급
                        DELETE  사용자 인증 토큰 만료
                        GET     사용자 인증 토큰 검증
                        
간단한 서비스 구현이 목표이므로 여기서는 RESTful 원칙에 약간 위배한 설계를 함
/users -> /users/<사용자번호> 가 더 나음
/users/<사용자번호> -> /users/<사용자번호>/token 가 더 나음
